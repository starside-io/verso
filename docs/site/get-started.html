Get started

Two ways to use Verso.

Pick the path that matches how you like to work. Either way it's the same files underneath, so you can switch any time.

Option A · friendliest

Visual editor

A web app with drag-and-drop, live preview, and one-click export. Great if you'd rather click than type.

npm install -g @starside-io/verso-cli
verso init my-deck --template layouts-gallery
cd my-deck
verso edit

The editor opens in your browser at http://localhost:5180. Save changes and the preview reloads.

Option B · keyboard-first

CLI + JSON files

Edit slide files in your favorite editor. Run verso dev to see them render. Diff in git, commit, ship.

npm install -g @starside-io/verso-cli
verso init my-deck --template minimal
cd my-deck
verso dev          # viewer at http://localhost:5173

Open slides/ in VS Code or any text editor. Saves auto-reload the viewer.

Templates

Don't start from a blank file.

verso init ships with seed decks. Pass --template <name> to pick one.

✦

minimal

One cover slide, one content slide. The smallest valid deck. Good for learning the schema.

▦

layouts-gallery

One slide per built-in layout. Browse what's possible, copy what you like.

●

multi-path

Demonstrates path branching with sales / engineering / exec audiences. The best starting point if branching is why you're here.

✓

branded

Custom theme + watermark + variables wired up. Strip and replace with your brand.

After you install

A 5-minute tour.

1. Add a slide

verso new slide my-first # or hit "+ Slide" in the editor

2. Tag it for an audience

// in slides/my-first.json
{
  "id": "my-first",
  "layout": "two-col",
  "path_include": ["sales"],
  "content": [ /* heading, bullets, etc. */ ]
}

3. Preview a path

open 'http://localhost:5173/?path=sales'

4. Export

verso build --format pdf
verso build --format html   # single self-contained file
verso build --format png    # per-slide images

5. Want something specific? Read the full docs or jump to how the schema works.

Before you install

A few small things.

⌨

Node 18+

Verso uses pnpm-style modern tooling. Any recent Node release works. Check with node -v.

✦

Browser

The editor and viewer run in any current Chrome, Firefox, Safari, or Edge. PDF export uses bundled headless Chromium.

●

No account, no cloud

Everything runs on your laptop. Files live on your disk. Sync with git, Dropbox, whatever you already use.

⤓

Apache 2.0

Use it commercially, fork it, ship a SaaS on top. No strings.

You're three commands away.
